# 《从数据结构到简单分析流程》修订大纲

## 修订定位

本节定位为进入 pandas 之前的桥接章节。它用“学生信息管理”这个例子，把已经学过的字典、列表、条件判断、循环、推导式和函数组织成一个简单的数据处理流程。

本节的讲解部分保持轻量，重点讲清楚三个问题：

1. 如何用 `dict` 表示一条记录，用 `List[dict]` 表示一组记录。
2. 如何围绕一个需求完成搜索、筛选、统计，并把常见操作提炼成函数。
3. 如何找到某一条记录，并添加或修改其中的字段。

## 章节结构

1. 开篇说明
   - 本章把已有知识放在一个小任务中使用。
   - 本章使用基础 Python 数据结构处理数据。
   - 后续 pandas 会提供更简洁的写法，本章保留分析思路。

2. 数据流
   - 用“上游数据 -> 处理步骤 -> 下游结果”理解程序。
   - 函数第一个参数通常放数据，后面的参数放处理条件。
   - 函数把任务拆成有输入、有输出的小步骤。

3. 从数据开始
   - 用表格建立阅读直觉。
   - 一位学生用 `dict` 表示。
   - 多位学生用 `List[dict]` 表示。
   - 数据量保持在 6 位学生，足够支持筛选和统计。

4. 添加功能：查找
   - 需求：找到某位叫 Alex 的学生。
   - 先用普通循环写筛选。
   - 再展示列表推导式。
   - 把按姓名查找提炼成函数。
   - 按学号查找只在讲解中提示，具体实现放到综合练习。

5. 添加功能：简单统计
   - 需求：统计金融专业有多少人。
   - 先筛选，再用 `len()` 计数。
   - 把按专业筛选和按专业计数提炼成函数。

6. 最小修改范例
   - 需求：给某位学生记录 Python 课成绩。
   - 先找到学生。
   - 再给该学生字典添加 `python_score`。
   - 说明添加字段和修改字段都使用字典赋值。
   - 按学号记录成绩的函数放到综合练习。

7. 综合练习
   - 练习承接讲解部分，不新增语法。
   - 覆盖：按学号查找、添加成绩、批量更新成绩、平均分、评级、及格状态、结果输出。
   - 练习采用函数框架和提示，学生在练习中完成串联。

## 修改重点

- 保留原版讲解骨架。
- 讲解部分只增加一个“修改数据”的最小范例。
- 普通循环先于列表推导式。
- 复杂分析任务放到综合练习中完成。
- 正面论述本章定位，使用直接说明式表达。
